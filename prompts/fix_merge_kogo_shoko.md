# 甲号証 結合機能 修正プロンプト（Claude Code 用）

このプロンプトは、訴訟用「甲号証管理アプリ」の **結合機能のみ** を修正するために
Claude Code に与える指示書である。コーディング以外の追加機能は実装しないこと。

---

## 1. 前提となるフォルダ／ファイル構成

ルートフォルダ直下に次のものが存在することを前提とする。**無ければ空のものを作成する**こと。

| 名称              | 種別        | 役割                                                                 |
|-------------------|-------------|----------------------------------------------------------------------|
| `甲号証リスト.docx` | Word ファイル | その申立書で使用する甲号証の番号（3桁、枝番あり可）が縦に並んだリスト |
| `個別マスタ/`     | フォルダ    | 各甲号証 docx の保管場所（`甲第ＮＮＮ号証.docx` などのファイル名）   |
| `結合甲号証/`     | フォルダ    | 結合後の `結合甲号証.docx` を保存する場所                             |

実行開始時にルートフォルダを走査し、上記が存在しなければ次を行う。

```text
- 甲号証リスト.docx が無い   → 中身が空の docx を新規作成
- 個別マスタ/ が無い          → 空フォルダを作成
- 結合甲号証/ が無い          → 空フォルダを作成
```

`甲号証リスト.docx` が **存在し、かつ番号が 1 件以上記載されている** 場合のみ、
そのリストを結合対象として用いる。それ以外（無い／空／番号が抽出できない）は
**`個別マスタ/` 配下の全 docx を対象** とする。

---

## 2. 使用するスクリプトと仕様書

修正・参照する対象は次の 2 ファイルに限定する。新規にロジックを書き直さないこと。

- `merge_kogo_shoko_仕様書.md` — 結合ロジックの仕様書
- `merge_kogo_shoko.py`        — 仕様書に基づく Python 実装

これらが存在しない場合は、本プロンプト末尾の参考実装に従って配置してよい。
存在する場合は、本修正プロンプトの要件と**齟齬がある箇所のみ**を修正する。

---

## 3. 結合フロー（今回の修正対象）

1. ルートフォルダを CLI 引数または起動ディレクトリから決定する。
2. 上記「前提構成」のチェック・自動生成を行う。
3. `甲号証リスト.docx` を `python-docx` で読み込み、各段落から番号を抽出する。
   - 抽出は本文用と同じ正規表現（後述）を使う。
   - 主番号 + 枝番のタプル `(main:int, branch:int|0)` の集合を作る。
4. リストが空でなければ → `個別マスタ/` から **そのタプルに一致するファイルだけ** を選ぶ。
   リストが空／無し → `個別マスタ/` 配下の全 docx（先頭 `~$` の一時ファイルを除く）を対象とする。
5. 対象ファイルを `merge_kogo_shoko.py` のロジック（仕様書 §「結合処理」）に渡し、
   番号順にソート・本文マーカーの正規化・改ページ挿入・`docxcompose` での結合を行う。
6. 出力は `結合甲号証/結合甲号証.docx`。同名ファイルがある場合は上書き前に確認するか、
   タイムスタンプ付きでバックアップしてから上書きする。

> **注意**：今回の修正範囲はあくまで「リストの有無で結合対象を切り替える」「番号正規化を
> 確実にする」「枝番を取りこぼさない」の 3 点。分解機能・一覧テーブル生成・GUI 等は
> 触らない。

---

## 4. 番号正規化ルール（厳守）

入力 docx 本文・ファイル名・甲号証リストのいずれにも、以下のような表記ゆれが出現する。

```text
甲第 1号証
甲第０1号証
甲第 12号証
甲第１２号証
甲01号証
甲第1号証
甲第０１２号証その１
甲第０１２号証その３
```

これらを次のように **すべて全角3桁ゼロ埋め** に統一する。

| 入力                      | 正規化後                |
|---------------------------|-------------------------|
| 甲第1号証                 | 甲第００１号証          |
| 甲第 12号証               | 甲第０１２号証          |
| 甲01号証                  | 甲第００１号証          |
| 甲第０１２号証その１     | 甲第０１２号証その１ |
| 甲第０１２号証その3       | 甲第０１２号証その３ |

ルール詳細：

- 主番号は **全角数字 3 桁** にゼロ埋め（`1 → ００１`、`12 → ０１２`、`123 → １２３`）。
  4 桁以上はゼロ埋めせずそのまま全角化。
- 「第」の有無、半角／全角数字、半角／全角スペースの混在は同一視する。
- 枝番は **全角数字（ゼロ埋めなし）** で表記する。
- 半角・全角混在を許容する抽出用の正規表現（目安）：

  ```text
  甲\s*第?\s*(?P<main>[0-9０-９]+)\s*号\s*証(?:\s*その\s*(?P<branch>[0-9０-９]+))?
  ```

- 抽出した main / branch は半角化して `int` に変換し、ソートキー `(main, branch or 0)` を作る。

正規化は次の 3 箇所すべてに適用する。

1. **甲号証リスト** から番号を取り出すとき
2. **個別マスタのファイル名** をリスト番号と突合するとき
3. **結合時、各 docx 本文先頭の【甲第…号証】マーカー** を書き換えるとき

ファイル名の正規化（個別マスタ側）は、**コピーまたはリネームで実体も統一する**こと。
書式維持のため、本文マーカーの書き換えは段落の最初のランにテキストを格納し、
他のランは空文字にする（仕様書「本文マーカーの書き換え」に従う）。

---

## 5. 枝番の取り扱い

- `甲第０１２号証その１` と `甲第０１２号証その３` は **別ファイル**として扱う。
- 主番号でグルーピングしたうえで、枝番昇順で並べる（`(012, 1) → (012, 3)`）。
- 甲号証リストに「012その1」のみ書かれていれば、`その3` は結合対象に **含めない**。
- 枝番の無いファイルとあるファイルが混在する場合、枝番無しを `branch=0` として
  先頭に並べる（リスト側に枝番無しの記載があった場合のみ対象）。

---

## 6. 改ページの必須化

ソート後、最後のファイル以外の末尾段落に改ページを追加する
（`run.add_break(WD_BREAK.PAGE)`）。最後のファイルには付けない。
これは `merge_kogo_shoko.py` の既存挙動でもあるため、**消さない・無効化しない**こと。

---

## 7. エッジケースと検証

- 重複番号（同じ `(main, branch)`）は警告し、最初のファイルを採用する。
- 先頭が `~$` の一時ファイルは除外。
- 文書冒頭にマーカーが見つからない docx は、書き換えをスキップして結合だけ続行する。
- 結合後に次を確認するログを出す：
  1. 各【甲第ＮＮＮ号証】マーカーが番号順か
  2. 各マーカーが新ページ先頭か（PDF 化で目視推奨）
  3. 表・画像・本文の欠落が無いか

---

## 8. 修正後にやること

1. ルートに `sample/` 等のテストフィクスチャを置き、次の 3 ケースを手動確認する：
   - 甲号証リストあり（一部だけ列挙）→ 列挙分だけ結合される
   - 甲号証リストあり（空）         → 個別マスタ全件が結合される
   - 甲号証リストなし               → 個別マスタ全件が結合される
2. `python merge_kogo_shoko.py <ルート>` で動作確認。
3. `claude/fix-exhibit-merge-n1amK` ブランチにコミット & プッシュ。
4. PR は **ユーザーの明示的な指示があるまで作成しない**。

---

## 9. やらないこと（スコープ外）

- GUI / Web UI の追加・変更
- 分解機能（結合甲号証 → 個別マスタ）への変更
- 甲号証一覧テーブル生成機能の変更
- 甲号証リスト自動作成機能の変更
- 既存の依存ライブラリ（`python-docx`, `docxcompose`）からの乗り換え

以上。
