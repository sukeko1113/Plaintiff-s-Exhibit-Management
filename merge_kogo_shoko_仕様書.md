# 甲号証ファイル結合ロジック仕様書（プロンプト版）

このドキュメントは、複数の「甲第xxx号証.docx」ファイルを番号順に1つに結合する
処理を、別のLLMアプリ（ChatGPT, Gemini, Claude API 等）でも再現できるよう
仕様化したものです。コード実装版は同梱の `merge_kogo_shoko.py` を参照してください。

────────────────────────────────────────────────────────────────────
■ コピー＆ペースト用プロンプト（LLMへの指示文）
────────────────────────────────────────────────────────────────────

あなたは日本の労働審判・訴訟における証拠書類（甲号証）を扱う書記補助です。
複数の「甲第xxx号証.docx」ファイルを以下のロジックで番号順に結合し、1つの
「結合甲号証.docx」を作成してください。

━━ 入力 ━━
複数の docx ファイル。各ファイルは冒頭に【甲第xxx号証】形式のマーカーを
持つ。ファイル名にも同形式の番号が含まれる。

━━ 番号正規化ルール ━━
1. 主番号は必ず「全角数字3桁」にゼロ埋めする。
   - 例: 1 → ００１ 、12 → ０１２ 、123 → １２３
   - 4桁以上の場合はゼロ埋めせずそのまま全角化（例: 1234 → １２３４）
2. 表記ゆれを吸収して同じ番号として扱う：
   - 「甲第 1号証」「甲第０1号証」「甲第１号証」「甲01号証」「甲第１号証」
     などはすべて「甲第００１号証」と同一視する。
   - 半角／全角数字、半角／全角スペース、「第」の有無 を許容する。
3. 枝番（「その１」「その2」等）に対応する：
   - 「甲第０１２号証その１」「甲第０１２号証その３」のように主番号が同じで
     枝番が異なる場合は、別ファイルとして扱う。
   - 枝番は全角数字（ゼロ埋めなし）で表記する。
   - 例: 甲第０１２号証その１ 、甲第０１２号証その２
4. 正規化マーカーの最終形式：
   - 枝番なし: 【甲第ＮＮＮ号証】
   - 枝番あり: 【甲第ＮＮＮ号証そのＭ】
   （Ｎ＝全角3桁、Ｍ＝全角枝番）

━━ 番号抽出（正規表現の目安）━━
本文・ファイル名の両方に対して以下のパターンでマッチさせる：

    甲\s*第?\s*(?P<main>[0-9０-９]+)\s*号\s*証(?:\s*その\s*(?P<branch>[0-9０-９]+))?

抽出した main / branch は半角化して整数に変換し、ソートキーとする
（タプル (main, branch or 0) で昇順ソート）。

━━ 番号取得の優先順位 ━━
1. 各ファイルを開き、冒頭から最大10段落以内に出現する【甲第…号証】マーカー
   を検出する。マーカー文字列がランをまたいで分割されている場合に備え、
   段落の text プロパティ（全ランを連結した文字列）に対して正規表現を適用する。
2. ファイル名からも番号を抽出し、本文と異なる場合は警告を出して本文側を優先する。
3. どちらからも抽出できない場合はエラーを表示し、そのファイルをスキップする。

━━ 本文マーカーの書き換え（正規化適用）━━
ソート後、結合前に各ファイルの本文先頭マーカーを正規化形式へ書き換える：
- 対象段落のテキストを正規化マーカー文字列（例: 「【甲第００１号証】」）で置換。
- 段落が複数のランに分割されている場合、最初のランに新テキストを格納し、
  残りのランのテキストを空にすることで書式（フォントサイズ・配置等）を維持する。
- 段落そのものは削除しない。

━━ 改ページ挿入（必須）━━
- 結合時、各甲号証の境界では必ず改ページを行い、次の甲号証は新ページ先頭から
  始まるようにする。
- 実装方法：ソート後の各ファイルのうち「最後のファイル以外」に対し、文書末尾
  段落へ改ページラン（python-docx であれば `run.add_break(WD_BREAK.PAGE)`）を
  追加する。最後のファイルには改ページを付けない。

━━ 結合処理 ━━
- 書式・画像・表・スタイルをできる限り保持するため、`docxcompose.composer.Composer`
  を使用する（`pip install docxcompose`）。
- 最初のファイルをベース Document として読み込み、残りのファイルを append する。
- ベースのスタイル・テーマが優先される。番号定義（list numbering）等は docxcompose
  が自動でリマップする。

━━ 出力 ━━
- 結合ファイル名: 「結合甲号証.docx」（指定がなければ）
- ファイル名正規化オプション: 入力ファイルのコピーを正規化済み名（例:
  「甲第００１号証.docx」「甲第０１２号証その１.docx」）で別フォルダに保存
  できるようにする。

━━ エッジケース ━━
- 重複番号: 同じ (main, branch) のファイルが複数あれば警告を出す。
- 一時ファイル（先頭が `~$`）は除外する。
- 文書冒頭にマーカーが見つからない場合：書き換えはスキップし、結合は続行する。
- ファイル3に大量の孤児画像（document.xml から参照されない media/ ファイル）が
  含まれる場合、結合時に削除されても本文の見た目は変わらないため問題ない
  （docxcompose の正常動作）。

━━ 検証手順 ━━
1. 結合後の docx を開き、各【甲第ＮＮＮ号証】マーカーが番号順に出現するか確認。
2. 各マーカーが新ページの先頭に配置されているか確認（PDF 化して目視推奨）。
3. 各甲号証の表・画像・本文が欠落していないか確認。

■ 擬似コード（言語非依存）
────────────────────────────────────────────────────────────────────

    function merge_kogo_shoko(input_files, output_path):
        # 1. 番号抽出
        items = []
        for f in input_files:
            num = extract_number_from_doc(f) or extract_number_from_filename(f)
            if num is None:
                warn(f, "番号抽出失敗")
                continue
            items.append((num, f))

        # 2. ソート
        items.sort(key = lambda x: (x[0].main, x[0].branch or 0))

        # 3. 各ファイルを一時コピーし、本文マーカーを正規化
        prepared = []
        for i, (num, f) in enumerate(items):
            tmp = copy_to_temp(f)
            rewrite_marker_in_first_paragraph(tmp, num.normalized_marker())
            if i < len(items) - 1:
                append_page_break_at_end(tmp)
            prepared.append(tmp)

        # 4. docxcompose で結合
        base = open_document(prepared[0])
        composer = Composer(base)
        for tmp in prepared[1:]:
            composer.append(open_document(tmp))
        composer.save(output_path)
