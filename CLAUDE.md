# プロジェクトルール

## UI保護(最重要)
- 既存のUIコンポーネント(JSX構造・className・レイアウト・色・余白)は変更禁止
- 機能追加でUI変更が必要な場合は、実装前に必ずユーザーに確認すること
- `v01-UI.md` をUI仕様の正本とする
- ボタンの配置・順序・ラベル文言の変更は禁止

## 開発方針
- 仕様は `SPEC.md` に従う
- 機能追加は既存ファイルを編集せず、新規ファイルで追加することを優先
- 変更前に「何をどのファイルでどう変えるか」を提案してから実装する

## 技術スタック
- Backend: Python / FastAPI / python-docx / docxcompose
- Frontend: React
- 出力形式: Markdown と Word(.docx)

## 禁止事項
- 実際の訴訟データフォルダ(Google Drive側)へのアクセス・参照は禁止
- 開発はこのフォルダ内で完結させること
