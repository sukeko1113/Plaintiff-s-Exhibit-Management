# 甲号証 結合プロンプト（Claude Code 用）

このプロンプトを Claude Code に貼り付けると、ルートフォルダ配下の甲号証ファイルを
仕様どおりに番号順で結合します。結合機能の修正のみが対象です（他機能は触らない）。

────────────────────────────────────────────────────────────────────
■ プロンプト本体（コピペして使う部分）
────────────────────────────────────────────────────────────────────

あなたは日本の労働審判・訴訟における甲号証（証拠書類）を扱う書記補助です。
今回のタスクは「甲号証ファイルの結合機能の修正」のみです。新機能の追加や
他の機能のリファクタリングは行わないでください。

## 0. 前提：使用するファイル

ルートフォルダ直下にある以下の2ファイルを必ず使ってください。
別実装を新規に作成しないでください（仕様の差分が出るため）。

- `merge_kogo_shoko_仕様書.md` — 結合ロジックの仕様書
- `merge_kogo_shoko.py` — 上記仕様の実装

実行前に以下を確認・準備してください。

```bash
python -c "import docx, docxcompose" 2>/dev/null || pip install python-docx docxcompose
```

## 1. フォルダ／ファイル構成の確認・作成

ルートフォルダ直下に以下が存在することを確認してください。
無いものは作成してください（フォルダは空のまま、ファイルは作らない）。

- `個別マスタ/`             — 各甲号証 docx の保管フォルダ（必須・無ければ作成）
- `結合甲号証/`             — 結合結果の出力フォルダ（必須・無ければ作成）
- `甲号証リスト.docx`       — 申立書で使用する甲号証番号を縦に並べた docx（任意）
                              ※ ファイルが無い／中身が空の場合は「全件結合」モードに切替

確認用のシェル例：

```bash
mkdir -p 個別マスタ 結合甲号証
ls -la 甲号証リスト.docx 2>/dev/null || echo "甲号証リスト.docx は存在しません（全件結合モード）"
ls 個別マスタ/*.docx 2>/dev/null | head
```

## 2. 結合対象の決定（重要：以下の条件分岐を厳守）

### (A) `甲号証リスト.docx` が存在し、かつ番号が1件以上抽出できる場合
リストに記載された番号のみを `個別マスタ/` から選んで結合します。

リストからの番号抽出は以下のルールで行います（正規表現の目安）：

```
甲\s*第?\s*(?P<main>[0-9０-９]+)\s*号?\s*証?(?:\s*その\s*(?P<branch>[0-9０-９]+))?
```

- 主番号のみの行（例: `001`, `12`, `１２３`）も許容してください
- 半角／全角数字、半角／全角スペース、「第」「号」「証」の有無を許容
- 枝番（「その１」「その2」「枝1」など）にも対応
- 重複行は排除し、`(主番号, 枝番)` のタプルでユニーク化
- `python-docx` で paragraph.text を順に走査して抽出

### (B) `甲号証リスト.docx` が無い／番号が抽出できない場合
`個別マスタ/*.docx` のすべてを結合対象にします。
標準出力に「全件結合モード」と明示してください。

### 共通の除外ルール
- ファイル名が `~$` で始まる Word の一時ファイルは必ず除外
- 番号が抽出できない docx はスキップ（標準エラーに警告を出力）

## 3. 番号正規化（仕様書のルールを必ず適用）

主番号は **全角3桁ゼロ埋め** に統一してください。

| 入力例 | 正規化後 |
| --- | --- |
| `甲第 1号証` | `【甲第００１号証】` |
| `甲第０1号証` | `【甲第００１号証】` |
| `甲第 12号証` | `【甲第０１２号証】` |
| `甲第１２号証` | `【甲第０１２号証】` |
| `甲01号証` | `【甲第００１号証】` |
| `甲第1号証` | `【甲第００１号証】` |
| `甲第０１２号証その１` | `【甲第０１２号証その１】` |
| `甲第０１２号証その３` | `【甲第０１２号証その３】` |

正規化対象は以下すべて：

1. ワード本文冒頭のマーカー（最大10段落以内を走査して書き換え）
2. ファイル名（コピー先ファイル名で正規化形式に揃える）
3. ソートキー（`(main, branch or 0)` の昇順）

枝番がある場合は **別ファイル** として扱い、主番号→枝番の順でソートします。

## 4. 結合処理の手順

`merge_kogo_shoko.py` を以下の流れで呼び出してください。

```bash
# 例：リスト指定モード（リストから抽出したファイル名のみ渡す）
python merge_kogo_shoko.py \
    "個別マスタ/甲第００１号証.docx" \
    "個別マスタ/甲第００２号証.docx" \
    "個別マスタ/甲第０１２号証その１.docx" \
    "個別マスタ/甲第０１２号証その２.docx" \
    -o "結合甲号証/結合甲号証.docx" \
    --rename --renamed-dir "結合甲号証/正規化済み"
```

```bash
# 例：全件結合モード（個別マスタの全 docx を一括処理）
python merge_kogo_shoko.py \
    --indir "個別マスタ" \
    -o "結合甲号証/結合甲号証.docx" \
    --rename --renamed-dir "結合甲号証/正規化済み"
```

スクリプトは内部で以下を行います（仕様書のとおり）：

1. 各 docx の冒頭マーカー／ファイル名から番号を抽出
2. `(main, branch)` 昇順でソート
3. 各ファイルを一時コピーし、本文先頭マーカーを正規化形式に書き換え
4. 最後以外のファイル末尾に改ページを挿入
5. `docxcompose.Composer` で結合し `結合甲号証/結合甲号証.docx` を出力

## 5. リスト→ファイル解決ロジック（実装メモ）

`甲号証リスト.docx` から得た `(main, branch)` を、`個別マスタ/*.docx` の各ファイルが
どの `(main, branch)` を表すかと突き合わせます。突き合わせは以下の手順で：

1. `個別マスタ` 内の各 docx について `merge_kogo_shoko.detect_number()` を呼んで
   `(main, branch)` を取得（本文マーカー優先・ファイル名フォールバック）
2. リスト側の `(main, branch)` と完全一致したものを採用
3. リストにあるが対応ファイルが見つからない番号は、標準エラーに警告を出力
   （結合は続行）
4. 個別マスタにあるがリストに無い番号は、リスト指定モードでは**結合に含めない**

リストの主番号のみの行（枝番なし）は、その主番号に紐づく**全枝番**（例: その１、その２…）を
含めて採用してください。`(main, None)` がリストにあれば、`個別マスタ` 内の同じ主番号の
docx をすべて拾います。

## 6. 動作確認

結合後、以下を確認して標準出力に簡潔に報告してください。

- 出力ファイル: `結合甲号証/結合甲号証.docx`
- 結合した甲号証の件数と番号一覧（`【甲第ＮＮＮ号証】` 形式）
- スキップ／警告の有無
- 「全件結合モード」か「リスト指定モード」のいずれだったか

PDF 化や目視確認まではこのプロンプトでは行いません。

## 7. やってはいけないこと

- `merge_kogo_shoko.py` の仕様外の改造（既存の正規化・改ページ・結合ロジックを変えない）
- `個別マスタ/` の元ファイル本体への書き換え（必ず一時コピーで処理する）
- 仕様書に無い新フォルダや新ファイルの作成（`結合甲号証/正規化済み/` は仕様内）
- 番号抽出失敗時のサイレント無視（必ず警告を出す）
- 文書末尾以外への余計な改ページ挿入
